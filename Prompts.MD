# Prompts

User-facing prompt templates for invoking actions from the trading agent.
Each numbered section = one type of action the user can trigger.
The workflow is not strictly linear ‚Äî pick the prompt that matches the user's intent at that moment.

---

## üåê Source Policy (global ‚Äî applies to every prompt that touches news, sentiment, or catalysts)

**Preferred sources, in priority order:**

1. **Primary sources** ‚Äî company IR pages, SEC filings (8-K, 10-Q, S-1), official press releases, conference-call transcripts.
2. **Reddit** ‚Äî ticker-specific subreddits first, then general retail subs:
   - Ticker-specific: `r/{TICKER}` if it exists (`r/NVDA`, `r/AMD_Stock`, `r/TSLA`, `r/GME`, etc.)
   - General: `r/wallstreetbets`, `r/stocks`, `r/options`, `r/investing`, `r/SecurityAnalysis`
3. **News wires** ‚Äî Reuters, Bloomberg, AP ‚Äî for confirmed events only (use sparingly).
4. **Social (named accounts only)** ‚Äî Twitter/X handles the user has flagged as KOLs. No anonymous bot-amplified posts.

**Excluded sources ‚Äî do not fetch, do not cite:**

- Seeking Alpha
- Motley Fool
- Zacks
- Investor's Business Daily op-eds
- Yahoo Finance / MarketWatch editorial content (raw price/quote data is fine; editorial articles are not)
- AI-generated content farms and SEO aggregators
- Any source whose business model is rage-driven engagement

**Rationale:** retail-driven subs surface flow and sentiment before aggregators repackage it. Primary sources are the ground truth. Aggregators add lag, recycle stale narratives, and pollute the catalyst signal. The agent wants signal, not SEO.

## Prompt 1 ‚Äî Trade Ideas

**Action:** Surface candidate trades for the user.

### 1.1 ‚Äî Watchlist-scoped one-cycle swing ideas (Stocks)

**Use when:** the user is out of original trade ideas and wants a templified list of candidates drawn from the existing watchlist.
**Scope:** stocks listed in `watchlist.MD` (this repo) / `watchlist.yaml` (tradingbot repo) only ‚Äî the agent should NOT pull from the broader market.
**Horizon:** one standard monthly options cycle (~19‚Äì22 trading days, ~21 on average).

**Default output shape ‚Äî 9 stocks, full-spectrum tier coverage, ranked by composite score:**

- **3 from Tier 1A** (mega-cap quality ‚Äî NVDA, AMD, GOOGL, META, AAPL, MSFT, AMZN, TSLA, SPY, QQQ)
- **3 from Tier 1B** (narrative growth, auto-pass numeric filters)
- **3 from Tier 1C** (override candidates ‚Äî top 3 of the moment, each respecting `watchlist.MD` ¬ß2 override discipline)

The user wants the full opportunity surface, not just the safest end. The 9-name list gives 3-deep coverage of each tier; the composite score lets the user see *which 1B beats which 1A* at any given moment.

#### Composite scoring ‚Äî 60% quant / 40% qual (0‚Äì100 scale)

**Composite = 0.60 √ó Quant + 0.40 √ó Qual**

Weighting choice: rules-based (quant) carries the heavier weight because the user's stated failure mode is emotional override ‚Äî anchoring more of the rank in ¬ß7's screener output protects against the "I like this name" drift. Qualitative still matters because catalysts, narrative momentum, and setup cleanliness aren't fully captured numerically.

**Quant Score (0‚Äì100)** ‚Äî fully rules-based, derived from `Trading Strategy.MD ¬ß7`:

| Component | Max points | Source |
|---|---|---|
| Screener grade (post-style-multiplier) | 60 | ¬ß7 screener: A = 60, B = 35, C = 10 (no C/D should reach this list) |
| R:R for the one-cycle hold | 20 | ‚â•3.0 = 20 ¬∑ 2.5 = 16 ¬∑ 2.0 = 12 ¬∑ 1.5 = 8 ¬∑ <1.5 = 0 |
| Distance-to-invalidation (entry ‚Üí stop, % of price) | 10 | tighter = better; ‚â§3% = 10, 3‚Äì5% = 7, 5‚Äì8% = 4, >8% = 0 |
| Liquidity headroom (ADV vs typical position) | 10 | comfortable = 10, adequate = 5, tight = 0 |

**Qual Score (0‚Äì100)** ‚Äî agent's structured judgment across four lenses:

| Component | Max points | What the agent is reading |
|---|---|---|
| Catalyst quality (next 19‚Äì22 days) | 30 | Upcoming earnings / product / macro inside the window ‚Äî favorable, neutral, or hostile to the thesis |
| Narrative momentum | 25 | Is the story building or fading? Per Source Policy: Reddit ticker sub + general subs + named-account social |
| Setup cleanliness | 25 | Subjective TA read ‚Äî does the chart look like an A+ setup, or is it a B that the screener over-rewarded? |
| Conviction qualifier | 20 | Is this one of the best trades on the watchlist right now, or just the best in its tier? |

#### Ranking rule

The 9 ideas are ordered **top ‚Üí bottom by Composite Score**, across tiers. Rank 1 is the most actionable trade right now ‚Äî could be a 1A, 1B, or 1C. Rank 9 is still tradeable but lower priority.

Each idea includes: thesis, setup style + multiplier, trigger, invalidation, one-cycle target, R:R, Quant / Qual / Composite scores.

**Override:** if the user explicitly specifies a different count or tier mix in their prompt (e.g., "give me 3 ideas from Tier 1A only", "show 10 shorts", "skip Tier 1C this week"), the override wins. With no override, return 9 stocks 3/3/3 ranked by composite.

## Prompt 2 ‚Äî Market Analysis

**Action:** Pull a fresh, no-cache snapshot of a named symbol ‚Äî price, 24h volume, funding rate, open interest, key support/resistance, recent price chart, and any breaking news.
**Use when:** the user has a specific ticker in mind and wants the lay of the land before going deeper.
**Scope:** any tradeable symbol. Platform-aware ‚Äî the user names which data platform to source from (Futu, Webull, YahooFinance, etc.).
**Variables:** `{TICKER}` (e.g. `AMD`, `BTC`), `{PLATFORM}` (e.g. `Futu`, `Webull`, `YahooFinance`).

### 2.1 ‚Äî Fresh-fetch market snapshot

**Template (user pastes, substitutes `{TICKER}` and `{PLATFORM}`):**

> Market analysis of {TICKER} on {PLATFORM}. Do not reuse any previously cached prices or odds from this conversation. Fetch current price, 24h volume, funding rate, open interest, and key support/resistance levels. Chart the recent price action. Search for any breaking news or events affecting this asset right now ‚Äî **sources per Source Policy: Reddit-first (`r/{TICKER}` + general subs), then primary sources; no aggregators (Seeking Alpha / Motley Fool / Zacks).**

**Expected agent output:**

1. **Chart** ‚Äî 1-month price chart of the symbol (default horizon; user can request tighter/wider).
2. **Analysis** ‚Äî short trend/context read.
3. **Market stats** ‚Äî structured block:

4. **Breaking news / events** ‚Äî surfaces today's catalyst if one exists; explicitly says "none" if not.
5. **Re-cut offer** ‚Äî closes by offering tighter/different horizons.

## Prompt 7 ‚Äî Watchlist Expansion

**Action:** Add one or more new tickers to the watchlist universe, with full metadata, and confirm every downstream registry (Reddit URL registry, screener, etc.) picks up the new entries.

**Use when:** the user wants to expand the universe ‚Äî typically after research surfaces a new setup, sector rotation, or theme.

|---|---|---|

| `watchlist.MD` | Trades | Add a row in the appropriate theme table (human-readable mirror). |
| `reddit_overrides.yaml` | tradingbot | Only if the ticker's real retail community sub differs from the naive `r/{TICKER}` (e.g. AMD ‚Üí `r/AMD_Stock`). |

**Variables:**

- `{TICKER}` ‚Äî e.g. `MRVL`
- `{TIER}` ‚Äî `1A` / `1B` / `1C` per `watchlist.MD ¬ß1`
- `{THEME}` ‚Äî one of: `semi_ai`, `photonics`, `ai_software`, `crypto`, `space`, `defense`, `fintech`, `energy`, `hardware`, `healthcare`, `industrial`
- `{THESIS}` ‚Äî 1-2 sentence thesis
- `{EARNINGS_DATE}` (optional) ‚Äî ISO `YYYY-MM-DD`
- `{PATTERNS_TO_WATCH}` (optional) ‚Äî list e.g. `[flag_breakout, support_hold]`

### 7.1 ‚Äî Add new tickers (and propagate to all registries)

**Template (user pastes, fills in tickers):**

> Add the following ticker(s) to my watchlist:

> - Ticker: `{TICKER1}`, Tier: `{TIER}`, Theme: `{THEME}`, Thesis: `{THESIS}`
